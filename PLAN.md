# AMPS Rust FFI Plan

This document outlines the plan for creating a Rust FFI (Foreign Function Interface) for the AMPS (Advanced Message Processing System) C++ client library.

## 1. Overview

### 1.1 AMPS Client Library

The AMPS C/C++ client library (version 5.3.5.1) provides high-performance messaging capabilities including:

- **Publish/Subscribe messaging**: Send and receive messages on topics
- **State-of-the-World (SOW) queries**: Query cached message state
- **Bookmark subscriptions**: Replay from transaction logs
- **Delta messaging**: Send/receive only changed fields
- **High Availability**: Automatic failover and reconnection via HAClient
- **Message Queues**: Guaranteed message delivery
- **Multiple message formats**: JSON, XML, BSON, FIX, NVFIX, Protocol Buffers, etc.

### 1.2 Key Classes in C++ API

| Class | Purpose |
|-------|---------|
| `AMPS::Client` | Basic client connection to AMPS server |
| `AMPS::HAClient` | High-availability client with automatic reconnection |
| `AMPS::Message` | Represents messages sent/received |
| `AMPS::Command` | Builder for AMPS commands |
| `AMPS::MessageStream` | Iterator for synchronous message retrieval |
| `AMPS::AMPSException` | Base exception class |

### 1.3 Exception Hierarchy

All AMPS exceptions derive from `AMPS::AMPSException`:

```
AMPSException
├── ConnectionException
│   ├── AlreadyConnectedException
│   ├── AuthenticationException
│   ├── ConnectionRefusedException
│   ├── DisconnectedException
│   ├── NameInUseException
│   └── NotEntitledException
├── CommandException
│   ├── BadFilterException
│   ├── BadRegexTopicException
│   ├── BadSowKeyException
│   ├── InvalidTopicException
│   ├── PublishException
│   └── SubscriptionAlreadyExistsException
├── StoreException
│   └── PublishStoreGapException
└── TimedOutException
```

## 2. Architecture

### 2.1 Layered Design

```
┌─────────────────────────────────────────────────────────────┐
│                    Rust Safe API Layer                      │
│  (Idiomatic Rust wrappers, Result<T, AmpsError> returns)    │
└─────────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────────┐
│                  Rust FFI Bindings Layer                    │
│         (Auto-generated by bindgen from C headers)          │
└─────────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────────┐
│                    C Wrapper Layer (C++)                    │
│   (C++ wrapper that catches exceptions, converts to C)      │
│          Compiled as static/shared library                  │
└─────────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────────┐
│                   AMPS C++ Client Library                   │
│              (libamps.a / amps.lib)                         │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 Why a C++ Wrapper is Required

**Critical**: C++ exceptions cannot cross the FFI boundary safely into Rust. The wrapper must:

1. Catch all AMPS exceptions at the C++ level
2. Convert exceptions to error codes and error messages
3. Present a C-compatible API that Rust can bind to
4. Handle object lifetime management

### 3.2 Wrapper Implementation (amps_ffi.cpp)

Key implementation patterns:

```cpp
#include "amps_ffi.h"
#include <ampsplusplus.hpp>
#include <cstring>

// Helper macro to catch exceptions and populate error info
#define CATCH_AMPS_EXCEPTIONS(block) \
    try { \
        block; \
        return AMPS_FFI_OK; \
    } catch (const AMPS::ConnectionException& e) { \
        if (error) { \
            error->code = AMPS_FFI_ERROR_CONNECTION; \
            strncpy(error->message, e.what(), sizeof(error->message) - 1); \
            error->message[sizeof(error->message) - 1] = '\0'; \
        } \
        return AMPS_FFI_ERROR_CONNECTION; \
    } catch (const AMPS::AlreadyConnectedException& e) { \
        if (error) { \
            error->code = AMPS_FFI_ERROR_ALREADY_CONNECTED; \
            strncpy(error->message, e.what(), sizeof(error->message) - 1); \
            error->message[sizeof(error->message) - 1] = '\0'; \
        } \
        return AMPS_FFI_ERROR_ALREADY_CONNECTED; \
    } catch (const AMPS::AMPSException& e) { \
        if (error) { \
            error->code = AMPS_FFI_ERROR_UNKNOWN; \
            strncpy(error->message, e.what(), sizeof(error->message) - 1); \
            error->message[sizeof(error->message) - 1] = '\0'; \
        } \
        return AMPS_FFI_ERROR_UNKNOWN; \
    } catch (const std::exception& e) { \
        if (error) { \
            error->code = AMPS_FFI_ERROR_UNKNOWN; \
            strncpy(error->message, e.what(), sizeof(error->message) - 1); \
            error->message[sizeof(error->message) - 1] = '\0'; \
        } \
        return AMPS_FFI_ERROR_UNKNOWN; \
    } catch (...) { \
        if (error) { \
            error->code = AMPS_FFI_ERROR_UNKNOWN; \
            strcpy(error->message, "Unknown exception"); \
        } \
        return AMPS_FFI_ERROR_UNKNOWN; \
    }

// Implementation example
extern "C" {

amps_client_t amps_ffi_client_create(const char* client_name, amps_ffi_error_info_t* error) {
    try {
        AMPS::Client* client = new AMPS::Client(client_name ? client_name : "");
        return reinterpret_cast<amps_client_t>(client);
    } catch (const std::exception& e) {
        if (error) {
            error->code = AMPS_FFI_ERROR_UNKNOWN;
            strncpy(error->message, e.what(), sizeof(error->message) - 1);
        }
        return nullptr;
    }
}

void amps_ffi_client_destroy(amps_client_t client) {
    if (client) {
        delete reinterpret_cast<AMPS::Client*>(client);
    }
}

int amps_ffi_client_connect(amps_client_t client, const char* uri, amps_ffi_error_info_t* error) {
    if (!client || !uri) {
        if (error) {
            error->code = AMPS_FFI_ERROR_NULL_POINTER;
            strcpy(error->message, "Null pointer argument");
        }
        return AMPS_FFI_ERROR_NULL_POINTER;
    }
    
    AMPS::Client* cpp_client = reinterpret_cast<AMPS::Client*>(client);
    CATCH_AMPS_EXCEPTIONS(
        cpp_client->connect(uri)
    );
}

// ... additional implementations
}
```

```rust
        .generate()

    let out_path = PathBuf::from(env::var("OUT_DIR").unwrap());
    bindings
        .write_to_file(out_path.join("bindings.rs"))
        .expect("Couldn't write bindings!");
```

```rust
// src/client.rs
use crate::error::{AmpsError, AmpsResult};
use crate::message::Message;
use std::ffi::{CStr, CString};
use std::os::raw::{c_char, c_void};
use std::ptr;
use std::sync::Arc;

pub struct Client {
    inner: *mut ffi::amps_client_t,
}

// Mark Client as Send but NOT Sync (AMPS client is not thread-safe)
unsafe impl Send for Client {}

impl Client {
    pub fn new(client_name: &str) -> AmpsResult<Self> {
        let name = CString::new(client_name)?;
        let mut error_info = ffi::amps_ffi_error_info_t {
            code: ffi::amps_ffi_error_t_AMPS_FFI_OK,
            message: [0; 1024],
        };
        
        let handle = unsafe {
            ffi::amps_ffi_client_create(name.as_ptr(), &mut error_info)
        };
        
        if handle.is_null() {
            return Err(AmpsError::from(error_info));
        }
        
        Ok(Client { inner: handle })
    }
    
    pub fn connect(&mut self, uri: &str) -> AmpsResult<()> {
        let uri = CString::new(uri)?;
        let mut error_info = ffi::amps_ffi_error_info_t {
            code: ffi::amps_ffi_error_t_AMPS_FFI_OK,
            message: [0; 1024],
        };
        
        let result = unsafe {
            ffi::amps_ffi_client_connect(self.inner, uri.as_ptr(), &mut error_info)
        };
        
        if result != ffi::amps_ffi_error_t_AMPS_FFI_OK {
            return Err(AmpsError::from(error_info));
        }
        
        Ok(())
    }
    
    pub fn logon(&mut self, options: Option<&str>, timeout_ms: i32) -> AmpsResult<()> {
        let opts = options.map(CString::new).transpose()?;
        let mut error_info = ffi::amps_ffi_error_info_t {
            code: ffi::amps_ffi_error_t_AMPS_FFI_OK,
            message: [0; 1024],
        };
        
        let result = unsafe {
            ffi::amps_ffi_client_logon(
                self.inner,
                opts.as_ref().map(|s| s.as_ptr()).unwrap_or(ptr::null()),
                timeout_ms,
                &mut error_info,
            )
        };
        
        if result != ffi::amps_ffi_error_t_AMPS_FFI_OK {
            return Err(AmpsError::from(error_info));
        }
        
        Ok(())
    }
    
    pub fn publish(&mut self, topic: &str, data: &str) -> AmpsResult<u64> {
        self.publish_with_expiration(topic, data, 0)
    }
    
    pub fn publish_with_expiration(
        &mut self,
        topic: &str,
        data: &str,
        expiration_secs: u64,
    ) -> AmpsResult<u64> {
        let topic = CString::new(topic)?;
        let mut error_info = ffi::amps_ffi_error_info_t {
            code: ffi::amps_ffi_error_t_AMPS_FFI_OK,
            message: [0; 1024],
        };
        
        let seq = unsafe {
            ffi::amps_ffi_client_publish(
                self.inner,
                topic.as_ptr(),
                data.as_ptr() as *const c_char,
                data.len(),
                expiration_secs as c_ulong,
                &mut error_info,
            )
        };
        
        if error_info.code != ffi::amps_ffi_error_t_AMPS_FFI_OK {
            return Err(AmpsError::from(error_info));
        }
        
        Ok(seq)
    }
    
    pub fn subscribe<F>(&mut self, topic: &str, filter: Option<&str>, handler: F) -> AmpsResult<String>
    where
        F: FnMut(&Message) + Send + 'static,
    {
        // Implementation using callback trampoline
        todo!()
    }
    
    pub fn disconnect(&mut self) -> AmpsResult<()> {
        let mut error_info = ffi::amps_ffi_error_info_t {
            code: ffi::amps_ffi_error_t_AMPS_FFI_OK,
            message: [0; 1024],
        };
        
        let result = unsafe {
            ffi::amps_ffi_client_disconnect(self.inner, &mut error_info)
        };
        
        if result != ffi::amps_ffi_error_t_AMPS_FFI_OK {
            return Err(AmpsError::from(error_info));
        }
        
        Ok(())
    }
}

impl Drop for Client {
    fn drop(&mut self) {
        if !self.inner.is_null() {
            unsafe {
                ffi::amps_ffi_client_destroy(self.inner);
            }
        }
    }
}
```

## 7. Exception Handling Flow

### 7.1 Exception Propagation

```
┌─────────────────────────────────────────────────────────────┐
│  AMPS C++ throws AMPS::ConnectionException("Connection refused")
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  C++ Wrapper catches exception in catch block
│  → Populates amps_ffi_error_info_t
│  → Returns AMPS_FFI_ERROR_CONNECTION
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Rust FFI receives error code
│  → Converts to AmpsError::Connection
│  → Returns Result::Err(AmpsError)
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Rust application handles error with ? operator or match
└─────────────────────────────────────────────────────────────┘
```

### 7.2 Panic Safety

The C++ wrapper must ensure that:
1. All exceptions are caught (including `std::exception` and `...`)
2. No C++ objects leak when exceptions occur
3. The error message buffer is always null-terminated
4. The error code is always set, even for unknown exceptions

## 8. Build Instructions

### 8.1 Prerequisites

- Rust 1.70+ with Cargo
- C++ compiler (GCC 9+, Clang 10+, or MSVC 2019+)
- CMake 3.16+ or Make
- AMPS C++ Client library (download from https://crankuptheamps.com)
- Docker and Docker Compose (for testing)

### 8.2 Setup Steps

```bash
# 1. Clone repository
git clone <repo-url>
cd amps-rust-ffi

# 2. Download and extract AMPS client
mkdir -p amps-client
tar -xzf amps-c++-client-5.3.5.1.tar.gz -C amps-client --strip-components=1

# 3. Build C++ wrapper
mkdir -p c-wrapper/build
cd c-wrapper/build
cmake ..
make
# Or on Windows with MSVC:
# cmake .. -G "Visual Studio 17 2022"
# cmake --build . --config Release
cd ../..

# 4. Build Rust library
cargo build

# 5. Run tests
docker-compose -f tests/docker/docker-compose.yml up -d
cargo test
docker-compose -f tests/docker/docker-compose.yml down
```

### 8.3 Platform-Specific Notes

#### Linux
- Link against `libstdc++` or `libc++`
- May need `-fPIC` flag for static libraries

#### macOS
- Link against `libc++`
- May need to adjust rpath for dynamic libraries

#### Windows
- Use `extern "C"` consistently to avoid name mangling issues
- Link against `msvcrt` or appropriate runtime
- Be careful with string encoding (UTF-8 recommended)
